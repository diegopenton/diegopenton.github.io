Modular Algorithmic Trading Framework (Fall 2025 – Present)

This project is a modular algorithmic trading framework built in Python that I am actively developing. The system is designed around a “tree-trunk and branches” architecture where each branch represents a different predictive model, such as RSI-based decision trees, Bollinger band mean-reversion, GARCH volatility forecasting, and LSTM/GRU sequence models. These branches generate signals and confidence scores that are routed through a trunk ensemble and policy layer, which determines the final trading action and position sizing. I structure the framework with unit testing, backtesting, and walk-forward validation to make sure every component is evaluated out-of-sample, reducing the risk of overfitting. The project is not just about writing models—it is about building a full research pipeline that can be extended with new strategies and easily tested across different assets and time periods.

At the same time, I focus on the practical aspects of algorithmic trading research. The framework includes a backtesting engine that calculates key metrics like Sharpe ratio, maximum drawdown, and hit rate, giving me a way to judge strategies on both return and risk. Each branch is intentionally lightweight and interpretable so that it can be debugged quickly, but when combined they create an ensemble that adapts across market conditions. Beyond just producing trades, I also incorporate risk management through volatility-aware position sizing and regime filters that control when strategies should be active. The project continues to grow as I add new branches, refine ensemble weighting, and experiment with deep learning models, making it a hands-on exploration of how modular design principles can be applied to quantitative finance.

Database Systems Project – The Book Nook

For my Database Systems course project, I designed and implemented a relational database for The Book Nook, a fictional bookstore specializing in science fiction and fantasy. The store needed a system to manage its growing catalog of books and merchandise, track customer purchase histories, and generate accurate invoices. I created an entity–relationship model with subtypes for products, separating physical books, eBooks, and merchandise. From there, I built the schema, wrote SQL scripts for table creation, and implemented stored procedures, queries, and views to automate invoicing, update inventory in real time, and analyze sales data.

Working independently, I normalized the design to the third normal form (3NF) to ensure consistency and efficiency. I tested the database using realistic dummy data and evaluated its performance with indexing to speed up queries. The final solution provided a strong foundation for handling inventory, customer data, and transactions across both in-store and online sales. This project not only strengthened my SQL and database modeling skills but also gave me practical experience in solving real-world business challenges with structured data systems.
